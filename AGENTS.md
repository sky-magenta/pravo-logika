# Авторство и сопровождение скилла `pravo-logika`

Гид для работы **внутри этого репозитория** — прочитайте перед правкой `SKILL.md`, референсов или манифестов.

> Терминология (рус.): прилагательное для «agent / agentic» — **«агентный»**, не «агентский».

## Два слоя

- **Логический слой** — `references/logic/`. Перенесён из проекта `logika`, правится косметически. Это подпрограммы: силлогизм, понятия, суждения, индукция, ошибки, законы.
- **Правовой слой** — `references/*.md` верхнего уровня + `SKILL.md`. Основная зона разработки. Опирается на логический слой через маршрутизацию в `SKILL.md`.

Порядок разбора зашит в `SKILL.md`: сначала логическая форма (движок), затем квалификация и толкование (право), в конце — каталог юридических дефектов.

## Инварианты (не нарушать при правках)

1. **Не источник права.** Скилл не подставляет номера статей и позиции судов по памяти — ставит флаг `[требует проверки: норма / практика]`. Любой пример или инструкция, которые заставляют модель утверждать содержание нормы без источника, — ошибка.
2. **Право и факт раздельно.** Формальная/материальная истинность = вопрос права/вопрос факта; в отчёте это разные разделы.
3. **Без персональных данных.** В контенте скилла (примеры, референсы, README) — никаких реальных дел, клиентов, имён, внутренних материалов. Только условные и учебные примеры.
4. **Атрибуция.** При распространении сохраняется MIT-уведомление и ссылка на исходный `logika` (см. LICENSE).

## Разметка референсов

Каждый файл правового слоя: назначение (одна строка) + ссылки на смежные файлы + разделы + чек-лист. Статьи каталога дефектов — по шаблону в конце `legal-errors.md`.
