# AGENTS.md — гид для внесения изменений в `pravo-logika`

Прочитайте перед правкой `SKILL.md`, референсов, манифестов или тестов. Файл описывает,
как устроен репозиторий и какие правила нельзя нарушать при контрибьюте.

Это скилл и плагин Claude Code: он проверяет **форму** правового рассуждения на праве РФ
(субсумпция, толкование, однозначность договора) и **не является источником права**.

> Терминология (рус.): прилагательное для «agent / agentic» — **«агентный»**, не «агентский».

## Два слоя

- **Логический слой** — `references/logic/`. Перенесён из проекта `logika` (материал восходит
  к «Учебнику логики» Г. Челпанова), правится только косметически. Это подпрограммы:
  силлогизм, понятия, суждения, индукция, ошибки, законы мышления. Исключение —
  `logic/deontic.md` (деонтические модальности O/P/F): добавлен в этом проекте, не производен
  от `logika`, правится как обычный файл; при правке сохранять нейтральность (без правовой
  лексики, `p` / «действие a») и формат слоя (нумерованные разделы, финальные «Примеры разбора»).
- **Правовой слой** — `references/*.md` верхнего уровня + `SKILL.md`. Основная зона разработки;
  опирается на логический слой через маршрутизацию в `SKILL.md`.

Порядок разбора зашит в `SKILL.md`: сначала логическая форма (движок), затем квалификация и
толкование (право), в конце — каталог юридических дефектов.

## Инварианты (не нарушать при правках)

1. **Не проверяет право по существу.** Скилл не подставляет номера статей и позиции судов по
   памяти — ставит флаг `[требует проверки: норма / практика]`. Любой пример или инструкция,
   которые заставляют модель утверждать содержание нормы без источника, — ошибка.
2. **Право и факт раздельно.** Формальная / материальная истинность = вопрос права / вопрос
   факта; в отчёте это разные разделы.
3. **Без персональных данных.** В контенте (примеры, референсы, README, тесты) — никаких реальных
   дел, клиентов, имён, внутренних материалов. Только условные и учебные примеры.
4. **Атрибуция.** Сохраняется MIT-уведомление и ссылка на исходный `logika` (Pavel Rykov) и
   «Учебник логики» Челпанова (public domain) — см. `LICENSE` и `THIRD_PARTY_NOTICES.md`.

## Как править

- **Референс правового слоя:** назначение (одна строка) + ссылки на смежные файлы + разделы +
  чек-лист в конце. Стиль — сухой, российский правовой язык, без нейросетевых клише.
- **Новый дефект в каталоге** (`references/legal-errors.md`): по шаблону в конце файла — русское +
  латинское имя, затем «как выглядит / почему дефект (со ссылкой на нарушенное правило) / как
  исправить / условный пример».
- **Имя и версия.** Имя скилла — `pravo-logika` (поле `name` в `SKILL.md`). При содержательных
  изменениях бампайте `version` синхронно в `SKILL.md` и трёх манифестах (`.claude-plugin`,
  `.codex-plugin`, `.cursor-plugin`).
- **Структура плагина.** Внутри `.claude-plugin/` — только `plugin.json`. Маркетплейс семейства
  вынесен в отдельный репозиторий [`sky-magenta/damascus-ink-plugins`](https://github.com/sky-magenta/damascus-ink-plugins)
  (перечисляет все три скилла через github-источники) — здесь `marketplace.json` нет.
  `commands/`, `references/`, `SKILL.md` — в корне репозитория, не внутри `.claude-plugin/`.

## Тесты

Набор E2E — в `evals/` (источник истины — `build_cases.py`).

- После правки примеров: `python evals/build_cases.py` — перегенерирует `cases.jsonl`.
- Обязательно, офлайн: `python evals/validate_cases.py` — проверка структуры и покрытия (должно
  завершиться «OK»); годится и для CI (`pytest evals/validate_cases.py`).
- Полный прогон через модель: `python evals/run_evals.py` (нужен `ANTHROPIC_API_KEY`; `--dry-run`
  собирает промпты без сети).
- Новый тип дефекта — добавляйте примеры: хотя бы один `defect` и один `clean`, чтобы ловить и
  пропуск дефекта, и ложное срабатывание.
- Перед завершением правки плагина/манифестов: `claude plugin validate .`.

## Чего не делать

- Не утверждать реальные статьи или судебную практику как факт в контенте — только флаг к проверке.
- Не добавлять реальные дела, клиентов, персональные данные.
- Не ломать атрибуцию (`LICENSE`, `THIRD_PARTY_NOTICES.md`).
- Не класть `commands/`, `skills/`, `agents/` внутрь `.claude-plugin/`.
