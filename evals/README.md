# E2E-тесты pravo-logika (100 примеров)

Набор сквозных (end-to-end) тестов: скилл загружается в системный контекст модели,
каждому примеру подаётся вход, а ответ модели сверяется с ожиданием. Проверяет то,
ради чего скилл существует, — **находит ли он нужный дефект и не выдумывает ли дефекты
там, где рассуждение корректно**.

## Что внутри

| Файл | Назначение |
|---|---|
| `cases.jsonl` | 100 размеченных примеров (генерируется из `build_cases.py`) |
| `build_cases.py` | источник истины: примеры как Python-данные → пишет `cases.jsonl` |
| `validate_cases.py` | офлайн-проверка структуры и покрытия (без сети, годится для CI/pytest) |
| `run_evals.py` | реальный прогон через Claude API + грейдинг |
| `results.json` | отчёт последнего прогона (создаётся `run_evals.py`) |

## Устройство примера

```json
{"id":"A1","mode":"review","category":"A.подмена_квалификации","difficulty":"easy",
 "input":"Проверь: сделка совершена без нотариального удостоверения, значит, она мнимая.",
 "expect":"defect","defect_ru":"Подмена квалификации","defect_lat":"ignoratio elenchi",
 "aliases":["подмена квалификации","ignoratio elenchi","подмена тезиса","мнимость"],
 "why":"Отсутствие формы не является признаком мнимости."}
```

- **mode** — `review` / `clause-check` / `fix`.
- **expect** — `defect` (должен назвать дефект из `aliases`) или `clean` (не должен выдумать дефект).
- Все примеры **условны**: реальные нормы и дела не воспроизводятся (инвариант скилла).

## Покрытие (133 примера)

- **Режимы:** review (84), clause-check (44), fix (5).
- **Ожидание:** defect (112) / clean (21). Сложность: easy (30) / medium (75) / hard (28).
- **Разделы** (по первой букве id):
  - `A`–`F` — базовый каталог дефектов (`references/legal-errors.md`): квалификация, толкование, право/факт, доказывание, модальности, понятия;
  - `L` — общие логические ошибки на правовом материале (petitio principii, post hoc, a dicto, ad hominem…);
  - `K`, `Q` — типы неоднозначности clause-check (`Q` — хитрые кванторы и пороги: конфликт floor/cap, нахлёст диапазонов, единица срока);
  - `M` — корпоративные / M&A сценарии (MAC, контроль, эскроу, индемнити, преимущественное право, ответственность директора);
  - `H` — пограничные (hard): замаскированные дефекты (растянутый круг, порог под видом a fortiori, «и» под видом «или») **и ловушки на переусердствование** (валидное систематическое толкование, валидное a fortiori — их скилл не должен флагать);
  - `N` — корректные примеры; `X` — режим правки.
- **`clean`-примеры (21)** — субсумпция, a contrario при закрытом перечне, аналогия, презумпции, однозначные пороги/определения/кванторы: ловят галлюцинации дефектов.

## Быстрый старт

Офлайн-проверка набора (без ключа и сети):

```bash
python evals/validate_cases.py      # отчёт по структуре и покрытию
pytest evals/validate_cases.py      # то же как тест
```

Пересобрать `cases.jsonl` после правки `build_cases.py`:

```bash
python evals/build_cases.py
```

Реальный прогон (нужен ключ). Экспортируйте `ANTHROPIC_API_KEY` или войдите `ant auth login`:

```bash
python evals/run_evals.py --dry-run          # проверить сборку промптов без сети
python evals/run_evals.py --limit 5          # быстрый прогон 5 примеров
python evals/run_evals.py                     # весь набор, грейдер alias, модель claude-opus-4-8
python evals/run_evals.py --grader llm        # грейдинг вызовом-судьёй (точнее для clean)
python evals/run_evals.py --mode clause-check # только один режим
```

## Как считается «прошёл»

- **`--grader alias` (по умолчанию, быстрый и бесплатный):** для `defect` — ответ модели
  содержит любой из `aliases` / латинское имя; для `clean` — есть вердикт «корректна / однозначна».
- **`--grader llm` (точнее):** отдельный вызов-судья сверяет ответ с ожиданием по существу.
  Рекомендуется для `clean`-примеров: имя дефекта может звучать по-разному, а факт «дефект не выдуман»
  надёжнее оценивает судья, чем поиск подстрок.

## Стоимость и кэширование

Системный контекст (весь скилл, ~десятки тысяч токенов) **одинаков для всех запросов** и
кэшируется prompt caching'ом — платится по сути один раз, дальше идёт по цене чтения кэша.
Модель по умолчанию — `claude-opus-4-8`, adaptive thinking + `effort=high` (при старом SDK
параметры автоматически опускаются). Прогон всех 100 примеров — это 100 вызовов модели
(плюс ещё 100 при `--grader llm`); соизмеряйте с бюджетом, для быстрой проверки берите `--limit`.

## Замечание

Это оценка ФОРМЫ разбора: назван ли нужный дефект / не выдуман ли лишний. Юридическую
верность самих формулировок (нормы, практику) скилл не устанавливает и оценка не проверяет —
такие места скилл помечает флагом `[требует проверки]`.
