# pravo-logika · «Скилл логики для юриста»

[![ci](https://github.com/sky-magenta/pravo-logika/actions/workflows/ci.yml/badge.svg)](https://github.com/sky-magenta/pravo-logika/actions/workflows/ci.yml)

Скилл-агент для проверки **формы** правового рассуждения на материале права РФ. Разбирает юридическую аргументацию так, как логик разбирает силлогизм: следует ли квалификация из нормы и факта, правомерен ли переход при толковании, однозначна ли формулировка договора. Все референсы на русском.

Это надстройка над логическим движком: правовой слой поверх классической формальной логики. Логический слой перенесён из скилла [`logika`](https://github.com/EvilFreelancer/logika) (см. «Атрибуция»).

> Проект [Damascus Ink](https://damascus-ink.ru); автор — основатель платформы [Moire AI](https://moire-ai.tech). Лендинг: [`docs/index.html`](docs/index.html). Об авторе — в разделе [«Об авторе и проекте»](#об-авторе-и-проекте).

## Что делает

- **Ревью правовой аргументации** — разбор меморандума или позиции: структура, субсумпция, толкование, каталог юридико-логических дефектов. Только диагностика, текст не переписывается.
- **Правка** — переписывание с исправленной аргументацией, минимальными изменениями, с сохранением стиля автора.
- **Clause-check** — проверка договорной формулировки на логическую однозначность (определения, перечни, условия, союзы «и/или», пороги, кванторы, модальности).
- **Сравнение двух позиций** — карта спора двух встречных памяток (истец/ответчик, buy-side/sell-side): об одном ли спорят стороны, тип расхождения (право / факт / оценка), скрытые посылки, несущая точка. Материальную правоту не присуждает.
- **Разбор нормы** — текст статьи или положения как объект: гипотеза / диспозиция / санкция, кванторы, исключения, отсылки и неоднозначности в самой норме.

Дополнительно clause-check покрывает **сроки, условия и механику закрытия** (отлагательные/отменительные условия, порядок исполнения, signing/closing, survival) — см. [`references/time-and-conditions.md`](references/time-and-conditions.md).

Как это выглядит на живом тексте — сквозной разбор меморандума в [`docs/example.md`](docs/example.md); каталог дефектов — [`docs/cheatsheet.md`](docs/cheatsheet.md); готовые запросы — [`docs/prompts.md`](docs/prompts.md).

## Граница: не проверяет право

Скилл проверяет, **как** построено рассуждение, а не **какова** норма. Нормы и судебную практику он по памяти не подставляет: где вывод опирается на конкретную статью или позицию суда, ставится флаг `[требует проверки: норма / практика]`, и проверка остаётся за человеком. Факт скилл принимает как данность автора и фиксирует пробелы в доказывании отдельно от дефектов права.

## Установка

**Как папка-скилл** — скопируйте репозиторий в корень скиллов (имя папки должно совпадать с полем `name` в `SKILL.md`):

| Инструмент    | Путь                              |
| ------------- | --------------------------------- |
| Claude Code   | `~/.claude/skills/pravo-logika/`  |
| Cursor        | `~/.cursor/skills/pravo-logika/`  |
| OpenAI Codex  | `~/.codex/skills/pravo-logika/`   |

**Как плагин (Claude Code)** — репозиторий является маркетплейсом Claude Code:

```
/plugin marketplace add sky-magenta/pravo-logika
/plugin install pravo-logika@damascus-ink
```

Проверка — `/plugin`. Плагин подключает и скилл (`SKILL.md` + `references/`), и слэш-команды `/pravo-logika:review`, `/pravo-logika:clause-check`, `/pravo-logika:compare`, `/pravo-logika:norm`.

## Вызов

- Слэш-команда `/pravo-logika:review`, `/pravo-logika:clause-check`, `/pravo-logika:compare` или `/pravo-logika:norm` (готовые шаблоны — в [`docs/prompts.md`](docs/prompts.md)).
- Обычный запрос: «проверь правовую логику», «разбери меморандум», «проверь формулировку договора», «сравни две позиции», «разбери норму».
- Автоматически — агент может подгрузить скилл сам, если запрос совпадает с `description` в `SKILL.md`.

## Экосистема Damascus Ink

Семейство открытых скиллов для юриста (право РФ, один автор, MIT) — каждый закрывает свой слой работы над письменной позицией:

| Скилл | Слой | Статус |
| --- | --- | --- |
| **`pravo-logika`** | **Валидность**: следует ли вывод — субсумпция, толкование, однозначность договора | этот репозиторий |
| [`pravo-ritorika`](https://github.com/sky-magenta/pravo-ritorika) | **Убедительность**: примет ли адресат — линия спора, порядок аргументов, адаптация, стресс-тест | опубликован |
| `pravo-grammatika` | **Язык**: как написано — работа с самим юридическим письмом | на подходе |

Рабочий цикл над позицией: `/pravo-logika:review` → правка → `/pravo-ritorika:status` → `/pravo-ritorika:structure` → `/pravo-ritorika:stress-test`. Риторика валидность не проверяет — логические дефекты возвращает сюда флагом `[→ pravo-logika]`; логика не оценивает подачу — слои не пересекаются.

## Статус

Версия 0.5.0. Пять режимов: ревью, правка, clause-check, **сравнение двух позиций** (`references/compare.md`) и **разбор нормы** (`references/norm.md`, §1.5 — текст статьи как объект: гипотеза/диспозиция/санкция, кванторы, оговорки, санкция без референта, противоречивые оговорки). Clause-check дополнен **логикой сроков и условий** (`references/time-and-conditions.md`: отлагательные/отменительные условия, потестативность, круг в порядке исполнения, механика signing/closing, survival). Логический слой перенесён полностью и дополнен деонтикой (`logic/deontic.md`). Толкование развёрнуто в дерево способов (буквальное → систематическое → телеологическое → историческое). В `SKILL.md` есть раздел «Сдержанность» — порог придирчивости против ложных срабатываний. Есть набор E2E-тестов на 170 примерах (`evals/`) — все пять режимов, каталог дефектов, корпоративные / M&A / банкротные сценарии, многодефектные меморандумы и корректные примеры на отсутствие ложных срабатываний; CI прогоняет валидатор набора на каждый push. Каталог дефектов — [`docs/cheatsheet.md`](docs/cheatsheet.md), готовые запросы — [`docs/prompts.md`](docs/prompts.md).
